# Contributing to AI Safety Sri Lanka - Resources

Thanks for considering a contribution. This repo is the shared knowledge base of the AI Safety, Governance & Ethics Chapter - Sri Lanka. Every PR - from a single typo fix to a new working-group charter - is welcome.

## Quick start

1. **Fork** the repo.
2. **Branch** from `main`: `git checkout -b add-<short-description>`.
3. **Edit** in markdown. Match the tone and structure of nearby files.
4. **Submit** a PR with a clear title and a one-paragraph description of what you've added and why.

## What we accept

| Type | Where it goes | Notes |
|------|---------------|-------|
| New course / reading | `README.md` table or `Literacy/` / `Reading.List/` | Include a one-line description |
| Glossary term | `Local.Languages/Glossary/README.md` | Note alternatives considered |
| Event materials | `Community/Events/YYYY/Qn/<event>/` | Slides, talking points, recording links |
| Working group charter | `Community/Working.Groups/<wg>/charter.md` | Use the template |
| Project brief | `Tech.Security/Projects/<project>.md` | Use the template |
| Research paper | `Local.Academia/Research.Papers/` | BibTeX + 2-4 sentence annotation |
| Student project | `Local.Academia/Student.Projects/` | Use the template |
| University profile | `Local.Academia/Universities/` | Use the template |
| Misinfo incident | `Civil.Society/Misinformation.Deepfakes/` | Always link to a credible source - never propagate the deepfake itself |
| Policy submission | `Governance/Public.Consultations/` | Date the file `YYYY-MM-DD-topic.md` |

## What we don't accept

- Self-promotion without substantive content
- Material that propagates misinformation, including hosting deepfake media
- Content that violates the [Code of Conduct](./CODE_OF_CONDUCT.md)
- Translations machine-generated without human review (we use human-checked translations only)
- Unsourced claims about individuals or organisations

## Style notes

- Prefer prose tables for resource lists.
- Always include a date when listing time-sensitive material (events, papers, policy drafts).
- Link to the canonical primary source. Mirror PDFs into the repo only with permission and an attribution note.
- Sinhala (සිංහල) and Tamil (தமிழ்) entries should use Indic script as primary; provide Romanisation only where ambiguity arises.

## Review process

1. PRs are reviewed by a working-group lead or chapter steering member.
2. Substantive contributions (new sections, charters) require two approvals.
3. Typo / link fixes can be merged with one approval.
4. Disputes go to the steering group; decisions are recorded in the relevant `decision_log.md`.
