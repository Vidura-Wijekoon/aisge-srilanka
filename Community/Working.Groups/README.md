# Working Groups

Small, focused workstreams. Each WG runs on a 6-12 week cadence with a clear deliverable.

| WG | Lead | Deliverable | Status |
|----|------|-------------|--------|
| Policy | TBD | Section-by-section commentary on NAIS 2028 | forming |
| Evaluation | TBD | Sinhala/Tamil eval suite v0.1 | forming |
| Education | TBD | Localised study-circle curriculum | forming |
| Translations | TBD | AI-safety glossary (Sinhala/Tamil) | forming |
| Civil Society | TBD | Election-misinfo monitoring playbook | forming |

## How to start a WG

1. Open an issue with `[wg-proposal]` describing the problem and the deliverable.
2. Find at least one co-lead and three committed contributors.
3. Get sign-off from the chapter steering group.
4. Create `Working.Groups/<wg-name>/charter.md`.

## Charter template

```markdown
# WG Name

**Leads:** @x, @y
**Cadence:** weekly / bi-weekly / monthly
**Term:** start date - end date
**Deliverable:** one sentence

## Problem
## Approach
## Members
## Decision log
```
